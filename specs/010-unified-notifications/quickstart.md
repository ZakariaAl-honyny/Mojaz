# Quickstart: Unified Notification Service

## Overview
This service provides an asynchronous, multi-channel dispatch layer for all notifications generated by the Mojaz Platform.

## Triggering a Notification

```csharp
// Inject INotificationService into your Application Service:
public class AppointmentService : IAppointmentService
{
    private readonly INotificationService _notificationService;

    public AppointmentService(INotificationService notificationService)
    {
        _notificationService = notificationService;
    }

    public async Task ConfirmAppointmentAsync(int appointmentId)
    {
        // 1. Business Logic...

        // 2. Dispatch Notification (Backgrounds to Hangfire automatically)
        await _notificationService.SendAsync(new NotificationRequest
        {
            UserId = applicantId,
            EventType = NotificationEvent.AppointmentConfirmed,
            ApplicationId = applicationId,
            Payload = new Dictionary<string, string> 
            {
                { "Date", appointmentDate.ToString("yyyy-MM-dd") }
            }
        });
    }
}
```

## How It Works
1. `SendAsync` immediately stores an In-App notification in the `Notifications` table.
2. It then schedules `ISmsProvider`, `IEmailProvider`, and `IPushProvider` via `BackgroundJob.Enqueue()`.
3. Hangfire processors execute these external calls in the background, logging outcomes in `SmsLogs` and `EmailLogs`.
4. Users fetch updates via the `/api/v1/notifications/unread-count` polling endpoint on the frontend.
