## Summary

Bootstrap the Mojaz backend as a clean-architecture .NET 8 solution with five projects
(Domain в†’ Shared в†’ Application в†’ Infrastructure в†’ API), four test projects, all 21 domain
entities, base classes, global exception handling, JWT-ready Swagger, Serilog logging,
and Docker-based SQL Server for local development.

No MediatR / CQRS вЂ” direct service injection is used throughout for MVP simplicity.
All EF Core entity configuration uses Fluent API only (no Data Annotations).
Soft-delete is enforced globally via `HasQueryFilter`.

## Phase 0 вЂ” Research & Decisions

> See [research.md](./research.md) for full decision rationale.

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Service pattern | Direct injection (no MediatR) | MVP simplicity; CQRS adds overhead with no benefit at this stage |
| EF Core mapping | Fluent API only | Clean separation, no attributes polluting domain entities |
| Soft-delete | Global HasQueryFilter | Consistent, automatic вЂ” never leaks deleted rows |
| Repository | Generic IRepository\<T\> + optional entity-specific | Avoids copy-paste while allowing custom queries |
| Auth | JWT Bearer + Refresh Token | Stateless, scales horizontally; matches constitution |
| Background jobs | Hangfire with SQL Server storage | Reuses existing DB; persistent across restarts |
| Logging | Serilog with ConsoleAndFile sinks | Structured, enriched logs; easy to pipe to cloud later |
| PDF generation | QuestPDF | Free OSS, .NET native, no external process |

### Key Design Decisions

#### Base Class Hierarchy


All 21 entities inherit from the most appropriate base class (see data-model.md).

#### IRepository\<T\> Interface (in Domain)

```csharp
public interface IRepository<T> where T : BaseEntity
{
    Task<T?> GetByIdAsync(Guid id, CancellationToken ct = default);
    Task<IReadOnlyList<T>> GetAllAsync(CancellationToken ct = default);
    Task<IReadOnlyList<T>> FindAsync(Expression<Func<T, bool>> predicate, CancellationToken ct = default);
    Task<T> AddAsync(T entity, CancellationToken ct = default);
    void Update(T entity);
    void Remove(T entity);   // soft-delete sets IsDeleted=true in concrete impl
    Task<int> CountAsync(Expression<Func<T, bool>> predicate, CancellationToken ct = default);
    Task<bool> ExistsAsync(Expression<Func<T, bool>> predicate, CancellationToken ct = default);
}
```

#### IUnitOfWork Interface (in Domain)

```csharp
public interface IUnitOfWork : IDisposable
{
    IRepository<T> Repository<T>() where T : BaseEntity;
    Task<int> SaveChangesAsync(CancellationToken ct = default);
    Task BeginTransactionAsync(CancellationToken ct = default);
    Task CommitTransactionAsync(CancellationToken ct = default);
    Task RollbackTransactionAsync(CancellationToken ct = default);
}
```

#### Middleware Pipeline Order (Program.cs)

```csharp
// 1. Security headers (outermost вЂ” applied to ALL responses including errors)
app.UseSecurityHeaders();

// 2. HTTPS redirect
app.UseHttpsRedirection();

// 3. Global exception handler (catches all unhandled exceptions BEFORE routing)
app.UseMiddleware<GlobalExceptionHandlerMiddleware>();

// 4. Request logging (logs every request/response, after exception handler)
app.UseMiddleware<RequestLoggingMiddleware>();

// 5. CORS
app.UseCors("MojazPolicy");

// 6. Rate limiting
app.UseRateLimiter();

// 7. Authentication + Authorization
app.UseAuthentication();
app.UseAuthorization();

// 8. Audit log (after auth so we know WHO made the request)
app.UseMiddleware<AuditLogMiddleware>();

// 9. Swagger (dev only)
if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

// 10. Map controllers
app.MapControllers();

// 11. Health check endpoint
app.MapHealthChecks("/health");
```

## Complexity Tracking

| Item | Why Needed | Simpler Alternative Rejected Because |
|------|------------|--------------------------------------|
| 5 projects instead of 1 | Enforce architectural boundaries; Domain must have 0 deps | Single project cannot enforce package isolation at compile time |
| Generic Repository + UoW | Consistent data access, testable, no EF Core leaking into Application | Direct DbContext usage would violate layer isolation; impossible to unit-test Application without real DB |
| GlobalExceptionHandler middleware | Map all exceptions to ApiResponse\<T\>; hide stack traces in prod | Per-controller try-catch is brittle and misses unhandled exceptions |
| Hangfire for notifications | Async, persistent, retryable background jobs | Thread.Task.Run is volatile; in-process queue lost on crash |

| PRD Table | Plan Entity | Deviation | Rationale |
